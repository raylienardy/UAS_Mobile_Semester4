3. PaymentActivity.java (users_menu_activity_payment.xml) → halaman ringkasan pesanan dan pembayaran.

- **ItemPesananModel.java**  
  Model item pesanan yang digunakan untuk menyimpan ke Firebase, struktur serupa dengan PesananItem tetapi dengan atribut `qty`.

  Model data pesanan lengkap yang disimpan di Firebase (id, tenantId, customerId, meja, waktu, status, daftar ItemPesananModel, totalHarga).

- **NotificationModel.java**  
  Model data notifikasi untuk tenant (id, tenantId, text, waktu, status). Disimpan di node `notifications` Firebase.

- **HomeActivity.java**  
  Halaman utama user. Setelah pembayaran berhasil, activity ini dibuka dan stack activity sebelumnya dibersihkan.

- **SharedPreferences "FoodCourtGoPrefs"**  
  Menyimpan `userId` user yang sedang login, digunakan untuk mengisi `customerId` saat menyimpan pesanan.

- **Firebase Realtime Database**  
  - Node `pesanan/{pesananId}` → menyimpan data pesanan (PesananAdminModel).  
  - Node `notifications/{notifId}` → menyimpan notifikasi untuk tenant.  

- **users_menu_activity_payment.xml (layout)**  
  Layout halaman pembayaran, berisi:  
  - Navbar (tombol kembali, judul "Pembayaran")  
  - ScrollView dengan card ringkasan pesanan (`llPesananList`), total bayar  
  - Area QR code (gambar dummy `ic_qr_dummy`)  
  - Tombol "Bayar Sekarang"

- **item_pesanan.xml**  
  Layout kecil yang digunakan untuk setiap item pesanan dalam daftar. (Tidak disertakan dalam kode di atas, tetapi di-inflate dengan `R.layout.item_pesanan`)

- **Drawable / resource:**  
  - `ic_back_white` → ikon kembali  
  - `bg_card_tenant_info` → background card  
  - `ic_qr_dummy` → gambar QR code dummy  
  - `button_biru` → background tombol bayar

**Transisi:**
  - ivBack → kembali ke DetailTenantActivity.java (activity ini ditutup)  
  - btnBayar → dialog konfirmasi → setelah "Bayar": simpan ke Firebase → HomeActivity.java dengan flag CLEAR_TOP & NEW_TASK (kembali ke beranda, tumpukan activity dibersihkan)
